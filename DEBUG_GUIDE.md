# 调试指南 - 如何查看日志和错误信息

## 🔍 为什么终端（Terminal）没有输出？

**原因**：
- 这是一个**前端 React 应用**，运行在**浏览器**中
- 所有的 `console.log()` 和 `console.error()` 输出都在**浏览器控制台**中显示
- **终端（Terminal）** 只显示 Vite 开发服务器的启动信息，不会显示浏览器端的日志

## 📱 如何查看日志和错误？

### 方法 1: 浏览器开发者工具（推荐）

1. **打开浏览器开发者工具**
   - 按 `F12` 键
   - 或右键点击页面 → 选择"检查" / "Inspect"
   - 或按 `Ctrl + Shift + I` (Windows) / `Cmd + Option + I` (Mac)

2. **切换到 Console（控制台）标签**
   - 点击顶部的 "Console" 标签
   - 这里会显示所有的日志和错误

3. **查看上传日志**
   - 上传文件时，应该能看到：
     ```
     📤 开始批量上传，总共 X 个文件，最大并发数: 3
     📎 开始上传文件 1/X: 文件名.jpg
     创建上传任务: {...}
     上传状态变化: {state: 'running', progress: '0.00%', ...}
     ```

4. **查看错误信息**
   - 如果有错误，会显示红色错误信息
   - 点击错误可以查看详细堆栈信息

### 方法 2: 浏览器网络标签（Network）

1. 打开开发者工具（F12）
2. 切换到 **Network（网络）** 标签
3. 上传文件时，可以看到：
   - Firebase Storage 的上传请求
   - 请求状态（成功/失败）
   - 错误响应（如果有）

### 方法 3: 应用内错误显示

- 上传失败时，会在页面上显示错误信息
- 查看上传区域下方的红色错误提示框

## 🐛 常见问题排查

### 问题 1: 控制台没有任何输出

**可能原因**：
- 控制台被清空了
- 日志级别被过滤了
- 代码没有执行到日志位置

**解决方法**：
1. 确保控制台没有被过滤（检查过滤器设置）
2. 确保选择了 "All levels" 或 "Verbose"
3. 刷新页面重新尝试

### 问题 2: 只看到错误，没有详细日志

**解决方法**：
1. 展开错误信息（点击错误）
2. 查看 "Stack trace"（堆栈跟踪）
3. 检查是否有 "CORS" 或 "Network" 相关错误

### 问题 3: 上传失败但没有错误信息

**可能原因**：
- Firebase Storage 安全规则阻止
- 网络连接问题
- 文件太大

**解决方法**：
1. 检查浏览器控制台的 Network 标签
2. 查看失败的请求及其响应
3. 检查 Firebase Console → Storage → 规则

- **所有日志都在浏览器控制台**，不在终端
- 终端只显示 Vite 服务器的启动信息
- 如果上传失败，控制台会有详细的错误信息
- 使用 `console.error()` 输出的错误会显示为红色
